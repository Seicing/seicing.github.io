树木之类的东西则是选择独立uv，之前打算和地图共享同一个uv，被草泥马辱骂了




建模、修正以及放到游戏里实装这些就不废话，目前的话就记录一下几个比较重要的点
①.模型里面所有的面都一定要绑定到骨头里，不然会出现互相打架的情况，产生以下的效果：

没有绑定骨骼的树干和树叶本身90°垂直了，极其蛋疼，，，

②.不同种类的模型适用不同的导出方式，主要有几类:
1.物件模型类，导出的时候一定要把所有的贴图信息去掉，只保留材质信息
2.Tile场景类，导出之前要手动把uv格子调整到原来的96.5%（这个草泥马可能会处理）
3.动作类，只需要导出骨骼，这一点直接使用blender里面的默认设置“战撸”（具体其实就是图里这样剔除了多余的东西）


③.实装到unity的时候，都适用于新建材质后，材质指定特定的贴图，然后让模型对应的材质信息适用对应的贴图，ok
遇到动画不会动的场合下，是unity的版本更新造成的问题，它自动适用了新的动画，而要让单一循环的动画播放起来的话，使用旧版的legacy才能正常播放而不是默认的generic


④.把所有成功实装到unity里面的模型进行prefab化
具体的流程就是：
1.Model检查内部，配置缩放、材质（场景缩放1.0，物件缩放0.6）,这里强制性要求是这个比例的缩放，如果物件太大还要进一步缩小的，在prefab里面缩小！
2.Model拖动到场景，不要属于某个层级
3.Model拖动到Prefab目录下的正确位置，然后删掉场景的Model对象
4.点击Prefab，选择面板的Open Prefab，在里面进行例如Mesh碰撞判定之类的配置
配置完成后使用配置好的Prefab进行调用

最后还是记录一下vscode里面要用到的对git操作相关的命令吧，因为特殊缘故，原来使用的源树林檎炸裂了，到底为啥俺暂且蒙在鼓里
用vscode直接打开git文件夹，主要执行几个命令：
git add ：把修改过的文件登记到准备上存列表中
git commit -m '更新策划案机制'：引号里面写上这次提交的备注内容
git push：推送，输入id“Seicing”以及密码（有123的那个）后就开始上存
git pull：拉取别人的修改内容
以及注意开工之前最好先pull一发拉取最新的内容，免得两个人撞车引发更加可怕的林檎爆裂事件
